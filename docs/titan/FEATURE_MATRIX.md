Customer portal, AI insights, booking, payments, real (deployed) Cloudflare infrastructure, email delivery (Auth.js's Email provider logs instead of sending), enterprise SSO, organization-scoped query filtering for `GET /api/leads`/`GET /api/organizations`/`GET /api/assessments`/`GET /api/audit` (all still Platform-Administrator-only ‚Äî `SECURITY_GUIDE.md`'s "Known, accepted gaps"), a self-service way to grant the Platform Administrator role (provisioned via direct SQL only, `OPERATIONAL_RUNBOOK.md`). **Admin portal**: the application shell and one module (Dashboard) are real (EAP-1 Phase 1, above) ‚Äî but Lead Management, Assessment Management, Organization Management, User Management, an Audit Center, and an Operations Center are not built yet, deliberately (`ROADMAP.md`'s EAP-1 section); don't read the shell's existence as implying any of those six exist. Nothing above should be read as implying progress on anything not explicitly listed.

## Explicitly not yet features (don't add rows for these until they're real)

Customer portal, AI insights, booking, payments, real (deployed) Cloudflare infrastructure, email delivery (Auth.js's Email provider logs instead of sending), enterprise SSO, organization-scoped query filtering for `GET /api/leads`/`GET /api/leads/search`/`GET /api/organizations`/`GET /api/organizations/search`/`GET /api/assessments`/`GET /api/assessments/search`/`GET /api/audit` (all still Platform-Administrator-only — `SECURITY_GUIDE.md`'s "Known, accepted gaps"), a self-service way to grant the Platform Administrator role (provisioned via direct SQL only, `OPERATIONAL_RUNBOOK.md`), a real user directory/picker (assignment is "assign to me"/"unassign" only, `DECISION_LOG.md`'s EAP-2 entry), server-side saved views (saved filters are `localStorage`-backed, per-browser only), virtualized table rendering (not justified by real data volume), an assessment lifecycle/status workflow (an assessment is one immutable row per completed run — no draft/in-progress state exists to manage, `DECISION_LOG.md`'s EAP-3 entry), an evidence/attachment panel (no file-upload concept exists in this data model — the closest real analogue, free-text answers, is already in Assessment Results' question responses), organization search by risk level or assessment status (no cross-repository join exists anywhere in this codebase's architecture, and assessments have no status field to filter by — `DECISION_LOG.md`'s EAP-4 entry), organization-scoped delegation of Organization Administration (create/update/archive/restore stay Platform-Administrator-only, the same line drawn around every other cross-organization route — `ARCHITECTURE.md`'s EAP-4 section). **Admin portal**: the application shell, Dashboard (EAP-1), Lead Management (EAP-2), Assessment Management (EAP-3), and now Organization Management (EAP-4) are real — but User Management, an Audit Center, and an Operations Center are not built yet, deliberately (`ROADMAP.md`'s EAP section); don't read any of the first four's existence as implying any of those three exist. Nothing above should be read as implying progress on anything not explicitly listed.
